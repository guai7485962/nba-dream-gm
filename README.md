# NBA 夢之隊 GM — 本機版

把遊戲接上真實 NBA 數據，跑在你自己的 Windows PC 上。
資料每次手動執行抓取腳本更新，存進本機資料庫，遊戲開局時從本機伺服器讀取。

---

## 這包裡有什麼

```
nba-dream-gm/
├─ requirements.txt     要安裝的 Python 套件
├─ fetch_players.py     抓真實球員數據 → 寫進資料庫（你會定期執行這個）
├─ database.py          資料庫建立與存取（不用直接動它）
├─ ovr.py               OVR 評分與薪資反推公式（想調平衡時改這裡）
├─ server.py            本機伺服器（玩之前先啟動它）
├─ static/
│  └─ index.html        遊戲本體（已改成從伺服器讀真實數據）
└─ README.md            就是這份說明
```

資料來源說明：
- **球員數據**（得分、籃板等）來自 NBA 官方 stats 端點，透過 `nba_api` 套件取得，免費、準確。
- **薪資**：官方沒有公開 API，所以用 OVR 反推一個合理數值（不是真實薪資，是讓薪資帽機制能運作的遊戲化數字）。公式在 `ovr.py`，可自行調整。

## 一、第一次設定（只需做一次）

### 步驟 1：安裝 Python

1. 到 https://www.python.org/downloads/ 下載 Python（3.10 以上）。
2. 安裝時**務必勾選最下方的「Add Python to PATH」**，否則命令列找不到 python。
3. 裝完後開「命令提示字元」（按 Win 鍵打 `cmd` 開啟），輸入下面確認：
   ```
   python --version
   ```
   有印出版本號（例如 Python 3.12.x）就成功。

### 步驟 2：把這個資料夾放到好找的位置

例如放到 `C:\nba-dream-gm`。

### 步驟 3：安裝套件

在命令提示字元，先切到專案資料夾，再安裝套件：
```
cd C:\nba-dream-gm
pip install -r requirements.txt
```
第一次會下載幾個套件，等它跑完。

## 二、抓取球員數據

在專案資料夾下執行：
```
python fetch_players.py
```
這會抓本季 OVR 前 150 名球員，算好 OVR 與薪資，寫進 `players.db`（資料庫檔，會自動產生）。

想抓不同數量：
```
python fetch_players.py --top 80      抓前 80 名
python fetch_players.py --top 240     抓前 240 名（足夠 30 隊用）
```

換球季（新賽季開打後）：開 `fetch_players.py`，把最上面的 `SEASON = "2024-25"` 改成新球季。

> ⚠️ **抓取禮儀**：腳本在每次請求間有延遲，請勿改短或高頻重複執行，避免 IP 被 NBA 伺服器暫時封鎖。一天跑一兩次足矣。
>
> 如果抓取失敗（顯示 timeout 或拒絕），通常是暫時性的，過幾分鐘再試。NBA 端點偶爾會擋來自非瀏覽器的請求。

---

## 三、啟動伺服器、開始玩

### 在這台電腦玩

1. 在專案資料夾下執行：
   ```
   python server.py
   ```
2. 看到「伺服器啟動中」訊息後，打開瀏覽器進：
   ```
   http://localhost:8000
   ```
3. 遊戲就開了。標題列下方會顯示 🟢 表示成功讀到真實數據。
4. 不玩時，回到命令提示字元按 `Ctrl + C` 停止伺服器。

---

## 四、讓同網路的手機也能玩（在家、同一個 Wi-Fi）

### 步驟 1：查出這台 PC 的區網 IP

開命令提示字元，輸入：
```
ipconfig
```
找「無線區域網路介面卡 Wi-Fi」下的「IPv4 位址」，會像 `192.168.1.23`。記下來。

### 步驟 2：放行防火牆（只需做一次）

第一次啟動 `server.py` 時，Windows 可能跳出防火牆詢問 → 勾選「**私人網路**」允許。

若沒跳出來，用**系統管理員身分**開 PowerShell（右鍵開始鈕 → 終端機(系統管理員)），執行：
```
New-NetFirewallRule -DisplayName "NBA Game Server" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
```

### 步驟 3：手機連線

1. 確認**手機和 PC 連同一個 Wi-Fi**。
2. 確認 `server.py` 正在這台 PC 上執行。
3. 手機瀏覽器輸入（換成你查到的 IP）：
   ```
   http://192.168.1.23:8000
   ```
4. 連得到就能玩了。可用瀏覽器的「加入主畫面」做成桌面圖示。

> 玩的時候 PC 必須開著、伺服器必須在跑、手機要在同網路。關掉伺服器或關機，手機就連不到了。

## 五、常見問題

**Q：手機連不到（轉圈圈或逾時）**
依序檢查：① 用的是 PC 的區網 IP 不是 localhost？② `server.py` 還在跑？③ 手機和 PC 同一個 Wi-Fi？④ 防火牆放行了 8000 埠？⑤ 路由器有沒有開「AP 隔離」（家用通常沒有，公司/學校 Wi-Fi 常有，那種網路連不到是正常的）。

**Q：遊戲開了但標題列顯示 🟡 離線後備數據**
表示前端沒讀到 API。確認你是從 `http://localhost:8000`（伺服器）打開，而不是直接用滑鼠雙擊 index.html 檔案打開。直接開檔案時前端連不到 API，會用內建的 75 名球員後備資料（遊戲仍可玩，只是不是最新真實數據）。

**Q：`python` 指令找不到**
重裝 Python 並記得勾「Add Python to PATH」，或改用 `py` 代替 `python`。

**Q：抓取一直失敗**
NBA 端點偶爾擋非瀏覽器請求，這是已知狀況。等幾分鐘再試；或先用內建後備資料玩（直接啟動 server.py 即可，遊戲會用 index.html 內建的 75 名球員）。

**Q：「📡 同步真實行情」按鈕在本機版有用嗎？**
那顆按鈕原本是呼叫 Claude API 的功能，在本機 http 環境下可能會被瀏覽器擋。本機版的真實數據來源是改用 `fetch_players.py` 抓官方數據——這比那顆按鈕更準確完整。兩者定位不同：本機版靠抓取腳本更新數據庫，線上版靠那顆按鈕做即時行情。

## 六、想擴充到 30 隊？

目前遊戲前端是 8 隊架構（你 + 7 支 AI）。資料庫和抓取已經能支撐更多球員（`--top 240`）。要真的擴成 30 隊，前端的賽程、季後賽、隊伍清單都要改寫——那是下一階段的工程，跟我說一聲можем再一起做。現在這版先讓你用真實數據、在 8 隊架構下完整體驗到季後賽。

---

## 七、日常使用流程（設定完成後）

```
1. 偶爾更新數據：  python fetch_players.py
2. 每次要玩之前：  python server.py
3. 瀏覽器或手機開： http://localhost:8000  或  http://<PC的IP>:8000
4. 玩完停止伺服器： Ctrl + C
```
